# TASK

## 任务背景

这是一个非常基础的 Python TCP 服务练习题，适合刚接触真实项目结构的同学。

项目已经有基本结构，但还缺少一部分命令处理逻辑。你的任务是在理解现有代码的基础上，把它补完整。

## 你需要完成的内容

1. 使用模板仓库创建你自己的仓库。
2. clone 到本地。
3. 创建你自己的开发分支。
4. 运行测试，看看哪些地方没有通过。
5. 修改代码，让测试尽量通过。
6. 填写 `report.md`。
7. 最终通过 Pull Request 或仓库链接提交结果。

## 本题主要考察什么

- 你会不会阅读说明
- 你会不会运行 Python 项目
- 你会不会根据测试补全代码
- 你能不能完成一个基础功能修复
- 你能不能简单说明自己做了什么

## 修改要求

你至少需要完成：

- 1 个真实代码修复
- 1 次测试验证

如果你能把所有测试都修到通过，会更好。

## 运行方式

### 运行测试

```bash
pytest
```

### 运行服务

```bash
python server.py
```

## 注意事项

- 不要直接在 `main` 分支修改
- 不要整个项目重写
- 尽量在现有结构上补全
- 如果你没有使用 AI，也可以在 `ai-usage.md` 里写“本次没有使用 AI”
