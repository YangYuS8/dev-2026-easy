# TASK

## 考核背景

你将接手一个用于练习的半成品后端仓库。项目主题是一个基于 `FastAPI + PostgreSQL` 的任务服务。

这个仓库不是完整成品，里面保留了一些待修复的问题。你的目标不是重写项目，而是在理解现有代码的基础上，完成一次基础排查和修复。

这份 easy 版本更适合刚接触真实开发流程的同学。

## 你需要完成的任务

1. 从 GitHub 获取仓库到本地。
2. 创建你自己的开发分支。
3. 尝试启动项目。
   - 优先使用 `docker compose up --build`
   - 如果本机环境不方便，也可以先尝试本地运行或测试
4. 至少定位并修复 **1 个真实问题**。
   - 可以是环境问题
   - 可以是配置问题
   - 也可以是 Python 代码问题
5. 至少完成 **1 个 Python 相关修复**。
6. 填写 `report.md`。
7. 如果你使用了 AI，可以简单填写 `ai-usage.md`。
8. 最终通过 Pull Request 或仓库链接提交结果。

## Git 要求

- 必须从 GitHub 获取仓库。
- 不要直接在 `main` 分支开发。
- 需要创建自己的开发分支。
- commit 数量不做强制要求，但建议至少提交 1 到 2 次有意义的 commit。

## 启动要求

推荐先尝试：

```bash
docker compose up --build
```

如果你本机环境暂时不方便，也可以：

- 先按 README 尝试本地运行
- 或先运行测试，再记录你遇到的问题

重点不是一定要一次成功，而是你要能说明：

- 你尝试了什么
- 遇到了什么问题
- 你最后修了什么

## Python 修改要求

你至少完成 **1 个** Python 相关修复，范围包括但不限于：

- 参数校验
- 返回结构整理
- 状态更新逻辑补全
- 错误处理优化
- 测试补充或修正

如果你能完成第 2 个修复，可以作为加分项。

## AI 使用说明

- 可以使用 AI 或 Agent 辅助
- 如果使用了，请在 `ai-usage.md` 里简单记录：
  - 你问了什么
  - 对方给了什么建议
  - 你有没有采纳
- 不要求写很长，能看出你有自己的判断就行

## 文档要求

- 必须填写 `report.md`
- `ai-usage.md` 可以简写
- 内容基于你的真实操作过程

## 验证要求

至少记录 1 次你实际做过的验证，例如：

- `docker compose up --build`
- `pytest`
- 访问 `/health`
- 调用 `/tasks`

## 注意事项

- 不要直接在 `main` 分支改代码
- 不要整个项目重写
- 不要只改文档不改实际问题
- 不要求你一次把所有问题都修完

## 本题主要想考察什么

这道题主要考察：

- 你会不会阅读仓库说明
- 你会不会尝试启动项目
- 你能不能根据报错定位问题
- 你能不能完成至少一个真实修复
- 你能不能清楚说明自己做了什么
