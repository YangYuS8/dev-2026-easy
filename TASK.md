# TASK

## 考核背景

你将接手一个用于考核的半成品后端仓库。项目主题是一个基于 `FastAPI + PostgreSQL` 的任务服务。

本仓库已经有基础结构和部分实现，但仍保留了一些环境、配置、测试和业务逻辑缺口。你的目标不是重写项目，而是在理解现有代码的基础上完成修复、补充和验证。

## 你需要完成的任务

1. 从 GitHub 获取仓库到本地。
2. 创建你自己的开发分支。
3. 不允许直接在 `main` 分支上开发。
4. 使用 `docker compose up --build` 启动并检查项目。
5. 至少修复 1 个环境或配置问题。
6. 至少完成 2 个 Python 相关任务或 bug 修复。
7. 补充并提交 `report.md`。
8. 补充并提交 `ai-usage.md`。
9. 最终通过 Pull Request 提交你的结果。

## Git / GitHub 要求

- 必须从 GitHub 获取仓库，不要只复制代码片段。
- 必须创建自己的开发分支。
- 不允许直接在 `main` 分支开发。
- 至少提交 3 次有意义的 commit。
- commit message 需要能反映修改目的。
- 最终提交形式必须是 Pull Request。

## Docker / Docker Compose 要求

- 必须使用 `Docker Compose` 方式完成一次完整启动流程。
- 推荐命令：`docker compose up --build`。
- 需要自己定位并修复至少 1 个环境或配置相关问题。
- 请在 `report.md` 中记录你的排查过程、原因判断和修复方式。

## 排查方向提示

这是一个半成品项目，问题不一定只在 Python 代码里。

你需要综合判断的问题范围可能包括：

- 环境变量与配置模板
- `Docker Compose` 配置
- 文档与实现不一致
- 应用默认值与实际运行环境不一致
- 接口返回结构、参数校验、错误处理

请不要只盯着单个文件或单个报错。建议结合文档、配置、代码、日志和测试结果交叉判断，再决定修改方向。

## Python 功能修改要求

你至少需要完成 2 个 Python 相关任务或修复，范围包括但不限于：

- 参数校验
- 路由返回结构整理
- 状态更新逻辑补全
- 错误处理一致性
- 测试补充或修正

请避免只做非常表面的格式改动，确保你的修改对行为或可维护性有实际价值。

## AI / Agent 使用要求

- 至少使用一次 AI 或 Agent 辅助完成具体子任务。
- 在 `ai-usage.md` 中记录你的提问、得到的建议、是否采纳、为什么采纳或不采纳、以及你如何验证。
- 不要把 AI 输出当成最终答案直接提交，必须体现你自己的判断。

## 文档提交要求

- 必须填写 `report.md`。
- 必须填写 `ai-usage.md`。
- 内容需要基于你自己的实际操作和排查过程。

## 验证要求

- 至少说明一次启动验证和一次接口或测试验证。
- 如果你增加了测试，也请在 `report.md` 中记录。

- 你自己的开发分支历史
- 至少 3 次有意义的 commit
- 一份 Pull Request
- 已填写的 `report.md`
- 已填写的 `ai-usage.md`

## 注意事项

- 不要删除项目后整体重写。
