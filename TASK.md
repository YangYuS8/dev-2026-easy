# TASK

## 考核背景

你将接手一个用于练习的半成品后端仓库。项目主题是一个基于 `FastAPI + SQLite` 的任务服务。

这个仓库不是完整成品，里面保留了一些待修复的问题。你的目标不是重写项目，而是在理解现有代码的基础上，完成一次基础排查和修复。

这份 beginner 版本更适合刚接触真实开发流程的同学。

## 你需要完成的任务

1. 使用模板仓库创建你自己的仓库。
2. clone 你自己的仓库到本地。
3. 创建你自己的开发分支。
4. 尝试启动项目。
   - 推荐先本地运行
   - Docker Compose 可以作为可选方式
5. 至少定位并修复 **1 个真实问题**。
6. 至少完成 **1 个 Python 相关修复**。
7. 填写 `report.md`。
8. 最终通过 Pull Request 或仓库链接提交结果。

## Git 要求

- 必须从你自己的 GitHub 仓库开发。
- 不要直接在 `main` 分支改代码。
- 需要创建自己的开发分支。
- commit 数量不做强制要求。

## 启动要求

推荐先尝试本地运行：

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

如果你会用 Docker，也可以尝试：

```bash
docker compose up --build
```

## Python 修改要求

你至少完成 **1 个** Python 相关修复，范围包括但不限于：

- 参数校验
- 返回结构整理
- 状态更新逻辑补全
- 错误处理优化
- 测试补充或修正

## AI 使用说明

- 可以使用 AI 或 Agent 辅助
- 如果使用了，可以简单填写 `ai-usage.md`
- 如果没有使用，也可以直接写“本次没有使用 AI”

## 文档要求

- 必须填写 `report.md`
- `ai-usage.md` 可以简写
- 内容基于你的真实操作过程

## 验证要求

至少记录 1 次你实际做过的验证，例如：

- `pytest`
- 启动项目
- 访问 `/health`
- 访问 `/tasks`

## 注意事项

- 不要直接在 `main` 分支开发
- 不要整个项目重写
- 不要求你一次把所有问题都修完

## 本题主要想考察什么

- 你会不会阅读仓库说明
- 你会不会尝试运行项目
- 你能不能根据报错定位问题
- 你能不能完成至少一个真实修复
- 你能不能清楚说明自己做了什么
